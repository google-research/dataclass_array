# Design v2: SSE for Todos — Agent Manager v2

*Date: 2026-04-02*

## Goal

Replace the polling loop in the Todos frontend plugin (currently
`setInterval(fetchTodos, 10_000)`) with Server-Sent Events (SSE) pushed by the
sidecar. The client receives **typed, complete event payloads** and updates
local state directly — no refetch needed.

--------------------------------------------------------------------------------

## Event Protocol

We define a single SSE stream endpoint. Each message is a JSON-encoded
`TodoEvent`.

### Event Shape (shared contract between server and client)

```
data: {"type":"todo_created","todo":{...}}
data: {"type":"todo_updated","todo":{...}}
data: {"type":"todo_deleted","todoId":"<uuid>"}
```

Three event types — one per mutation class:

Type           | Payload         | Triggered by
-------------- | --------------- | ------------------------
`todo_created` | full `Todo`     | `POST /api/todos/todo`
`todo_updated` | full `Todo`     | `PATCH /api/todos/{id}`
`todo_deleted` | `todoId string` | `DELETE /api/todos/{id}`

The payload carries the **full updated object** (not a delta) so the client can
merge it into state directly. No refetch is needed.

--------------------------------------------------------------------------------

## Backend Design (Go)

### File layout — one feature per file

Following the "one feature per file" rule, the SSE feature lives in a
**dedicated file**, separate from the existing HTTP handler file.

```
plugins/todos/sidecar/
  types.go           ← Domain types (Todo, TodoAttempt, …) — EXISTING
  repo.go            ← DB queries                           — EXISTING
  plugin.go          ← HTTP handlers, route registration    — EXISTING (modified)
  hub.go             ← NEW: SSE Hub — subscriber management + broadcast
  events.go          ← NEW: TodoEvent types + JSON encoding
```

--------------------------------------------------------------------------------

### `events.go` — Typed event definitions

The event types are the **contract** between server and client. They live in
their own file so the shape is always easy to find and evolve independently.

```go
package todos

import "encoding/json"

// TodoEventType is the discriminant for a TodoEvent.
type TodoEventType string

const (
    EventTodoCreated TodoEventType = "todo_created"
    EventTodoUpdated TodoEventType = "todo_updated"
    EventTodoDeleted TodoEventType = "todo_deleted"
)

// TodoEvent is the SSE payload type.  Exactly one of Todo or TodoID is set,
// depending on the event type.
type TodoEvent struct {
    Type   TodoEventType `json:"type"`
    Todo   *Todo         `json:"todo,omitempty"`   // set for created / updated
    TodoID string        `json:"todoId,omitempty"` // set for deleted
}

// Encode serializes the event as a JSON string, for use in an SSE "data:" line.
func (e TodoEvent) Encode() (string, error) {
    b, err := json.Marshal(e)
    return string(b), err
}
```

**Design rationale:** - `TodoEvent` is a value type (struct, not pointer).
Immutable once constructed. - `Encode()` is placed on the type that owns the
data — not in the handler. - The event type discriminant uses a named
`TodoEventType` string alias so callers use constants (`EventTodoCreated`)
rather than raw strings.

### `hub.go` — SSE Hub

The Hub manages all open SSE connections. It is entirely isolated from HTTP and
database concerns. It does one thing: **fan-out broadcast to registered
channels**.

```go
package todos

import "log"

// Hub manages active SSE subscriber channels and broadcasts events to all of them.
//
// Usage:
//   h := NewHub()
//   go h.Run()  // must be started once in Plugin.Init
//
//   // In an SSE handler:
//   ch := h.Subscribe()
//   defer h.Unsubscribe(ch)
//
//   // In a mutation handler:
//   h.Broadcast(TodoEvent{Type: EventTodoUpdated, Todo: &updated})
type Hub struct {
    clients    map[chan TodoEvent]struct{}
    subscribe   chan chan TodoEvent
    unsubscribe chan chan TodoEvent
    broadcast   chan TodoEvent
}

// NewHub allocates a Hub. Call Run() in a goroutine before any use.
func NewHub() *Hub {
    return &Hub{
        clients:     make(map[chan TodoEvent]struct{}),
        subscribe:   make(chan chan TodoEvent),
        unsubscribe: make(chan chan TodoEvent),
        broadcast:   make(chan TodoEvent, 16),
    }
}

// Run is the Hub's event loop. It must run in its own goroutine.
func (h *Hub) Run() {
    for {
        select {
        case ch := <-h.subscribe:
            h.clients[ch] = struct{}{}
        case ch := <-h.unsubscribe:
            delete(h.clients, ch)
            close(ch)
        case event := <-h.broadcast:
            for ch := range h.clients {
                select {
                case ch <- event:
                default:
                    // Slow client: drop the event rather than blocking the loop.
                    log.Printf("todos hub: slow client, dropping event %s", event.Type)
                }
            }
        }
    }
}

// Subscribe registers a new client and returns its event channel.
func (h *Hub) Subscribe() chan TodoEvent {
    ch := make(chan TodoEvent, 8)
    h.subscribe <- ch
    return ch
}

// Unsubscribe removes the client and closes its channel.
func (h *Hub) Unsubscribe(ch chan TodoEvent) {
    h.unsubscribe <- ch
}

// Broadcast sends an event to all registered clients.
func (h *Hub) Broadcast(event TodoEvent) {
    h.broadcast <- event
}
```

**Design rationale:** - The Hub is a pure concurrency primitive — no HTTP, no
DB. - Its public API is three verbs: `Subscribe`, `Unsubscribe`, `Broadcast`. -
The channel capacity (8 per client, 16 broadcast buffer) prevents a single slow
client from stalling the broadcast loop. - `Run()` is the only goroutine; all
state lives inside it (no mutex needed).

### `plugin.go` — Changes to existing handlers

`Plugin` gains a `hub *Hub` field. Mutations call `h.hub.Broadcast(...)`
immediately after the DB write succeeds. The SSE endpoint is registered
alongside the existing routes.

#### Plugin struct and Init

```go
type Plugin struct {
    r   *repo
    hub *Hub  // ← NEW
}

func (p *Plugin) Init(ctx context.Context, pool *pgxpool.Pool) error {
    if _, err := pool.Exec(ctx, schemaSQL); err != nil {
        return fmt.Errorf("todos: apply schema: %w", err)
    }
    p.r = &repo{pool: pool}
    p.hub = NewHub()   // ← NEW
    go p.hub.Run()     // ← NEW
    return nil
}
```

#### RegisterRoutes

```go
func (p *Plugin) RegisterRoutes(mux *http.ServeMux) {
    mux.HandleFunc("/api/todos/todos",   p.handleTodos)
    mux.HandleFunc("/api/todos/attempts", p.handleAttempts)
    mux.HandleFunc("/api/todos/todo",    p.handleTodo)
    mux.HandleFunc("/api/todos/attempt", p.handleCreateAttempt)
    mux.HandleFunc("/api/todos/events",  p.handleEvents)  // ← NEW
    mux.HandleFunc("/api/todos/",        p.handleTodoByID)
}
```

#### Broadcast after mutations

Each mutation handler broadcasts **after** the successful DB write, passing the
operation result directly:

```go
// handleTodo — POST creates a todo
created, err := p.r.createTodo(...)
// ... error handling ...
p.hub.Broadcast(TodoEvent{Type: EventTodoCreated, Todo: &created})

// handlePatchTodo — PATCH updates a todo
updated, found, err := p.r.updateTodo(...)
// ... error handling ...
p.hub.Broadcast(TodoEvent{Type: EventTodoUpdated, Todo: &updated})

// handleDeleteTodo — DELETE removes a todo
p.hub.Broadcast(TodoEvent{Type: EventTodoDeleted, TodoID: id})
```

#### SSE handler

```go
// handleEvents implements GET /api/todos/events — SSE stream.
//
// The client subscribes once on mount and receives TodoEvents for every
// mutation, allowing the UI to update without polling.
func (p *Plugin) handleEvents(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodGet {
        w.WriteHeader(http.StatusMethodNotAllowed)
        return
    }

    flusher, ok := w.(http.Flusher)
    if !ok {
        http.Error(w, "streaming unsupported", http.StatusInternalServerError)
        return
    }

    w.Header().Set("Content-Type", "text/event-stream")
    w.Header().Set("Cache-Control", "no-cache")
    w.Header().Set("Connection", "keep-alive")

    ch := p.hub.Subscribe()
    defer p.hub.Unsubscribe(ch)

    // Send an initial "connected" keepalive comment so the browser knows the
    // stream is alive before any mutation events arrive.
    fmt.Fprintf(w, ": connected\n\n")
    flusher.Flush()

    for {
        select {
        case <-r.Context().Done():
            return
        case event := <-ch:
            encoded, err := event.Encode()
            if err != nil {
                log.Printf("todos: sse encode error: %v", err)
                continue
            }
            fmt.Fprintf(w, "data: %s\n\n", encoded)
            flusher.Flush()
        }
    }
}
```

**Note:** The SSE comment line (`: connected`) is a valid SSE keepalive that
doesn't trigger `onmessage` on the client — it just signals the stream is open.
This prevents the browser from immediately retrying the connection while waiting
for the first event.

## Frontend Design (TypeScript)

### File layout — one feature per file

```
plugins/todos/
  plugin.tsx                   ← Main plugin (modified)
  hooks/useTodosStream.ts      ← NEW: SSE hook — isolated, reusable
  components/
    TodoChatInput.tsx
    ...
```

The SSE lifecycle is extracted into a dedicated hook `useTodosStream`. The hook
signature is designed so `plugin.tsx` doesn't know anything about SSE internals:

```typescript
// hooks/useTodosStream.ts
useTodosStream(ws: string, onEvent: (event: TodoEvent) => void): void
```

--------------------------------------------------------------------------------

### `hooks/useTodosStream.ts` — SSE hook

```typescript
import {useEffect} from 'react';
import type {Todo} from '../plugin';  // or a shared types file

// --- Event types (mirror the Go contract) ---

export type TodoEventType = 'todo_created' | 'todo_updated' | 'todo_deleted';

export interface TodoCreatedEvent {
  type: 'todo_created';
  todo: Todo;
}
export interface TodoUpdatedEvent {
  type: 'todo_updated';
  todo: Todo;
}
export interface TodoDeletedEvent {
  type: 'todo_deleted';
  todoId: string;
}

export type TodoEvent = TodoCreatedEvent | TodoUpdatedEvent | TodoDeletedEvent;

// --- Hook ---

/**
 * Opens a Server-Sent Events connection to `/api/todos/events` and fires
 * `onEvent` for every received event. The connection is closed when the
 * component unmounts or `ws` changes.
 */
export function useTodosStream(
  ws: string,
  onEvent: (event: TodoEvent) => void,
): void {
  useEffect(() => {
    const source = new EventSource(`/${ws}/api/todos/events`);

    source.onmessage = (e: MessageEvent<string>) => {
      try {
        const event = JSON.parse(e.data) as TodoEvent;
        onEvent(event);
      } catch (err) {
        console.error('todos SSE: failed to parse event', e.data, err);
      }
    };

    source.onerror = () => {
      // EventSource auto-reconnects; the error is transient.
      console.warn('todos SSE: connection error (will retry)');
    };

    return () => {
      source.close();
    };
  }, [ws, onEvent]);
}
```

**Design rationale:** - `useTodosStream` knows nothing about component state —
it only calls `onEvent`. - The browser's built-in `EventSource` auto-reconnects
on failure — no manual retry logic needed. - Type-safe discriminated union
`TodoEvent` mirrors the Go `TodoEventType` constants. - `onEvent` is a plain
callback, not a state setter, so the caller decides how to apply the event.

### `plugin.tsx` — Changes to `TodosPane`

The `TodosPane` component replaces `setInterval` with `useTodosStream` and adds
an event handler that applies each event directly to local state:

```typescript
import {useTodosStream} from './hooks/useTodosStream';
import type {TodoEvent} from './hooks/useTodosStream';

function TodosPane() {
  const [todos, setTodos] = useState<Todo[]>([]);
  // ...

  // Derive ws once
  const currentWs = window.location.pathname.split('/')[1] || 'head';

  // Initial fetch on mount (SSE doesn't send current state on connect,
  // only future events — so we still need one initial GET).
  useEffect(() => {
    fetchTodos();
  }, [fetchTodos]);

  // Apply incoming SSE events directly to local state — no refetch.
  const handleTodoEvent = useCallback((event: TodoEvent) => {
    switch (event.type) {
      case 'todo_created':
        setTodos(prev => [...prev, event.todo]);
        break;
      case 'todo_updated':
        setTodos(prev =>
          prev.map(t => (t.id === event.todo.id ? event.todo : t)),
        );
        break;
      case 'todo_deleted':
        setTodos(prev => prev.filter(t => t.id !== event.todoId));
        // Clear selection if the selected todo was deleted.
        setSelectedId(prev => (prev === event.todoId ? undefined : prev));
        break;
    }
  }, []);

  // Open SSE stream. Auto-reconnects on error.
  useTodosStream(currentWs, handleTodoEvent);

  // ... rest of component unchanged ...
}
```

**The toolbar "Refresh" button is kept** as a manual escape hatch (e.g. if the
SSE stream reconnects after a server restart and misses events during downtime).

## Abstraction Boundaries Summary

```
┌─────────────────────────────────────────────────────────────────┐
│  plugin.tsx (TodosPane)                                         │
│  • calls useTodosStream(ws, handleTodoEvent)                    │
│  • handleTodoEvent applies typed events to useState             │
│  • knows nothing about EventSource, reconnect, SSE format       │
└────────────────────┬────────────────────────────────────────────┘
                     │ callback (TodoEvent)
┌────────────────────▼────────────────────────────────────────────┐
│  hooks/useTodosStream.ts                                        │
│  • owns EventSource lifecycle (open / close / error)           │
│  • parses JSON into typed TodoEvent                             │
│  • knows nothing about React state or component tree            │
└────────────────────┬────────────────────────────────────────────┘
                     │ SSE text/event-stream
┌────────────────────▼────────────────────────────────────────────┐
│  plugin.go (handleEvents)                                       │
│  • handles HTTP SSE protocol (headers, flush, keepalive)        │
│  • delegates to Hub for subscription lifecycle                  │
│  • knows nothing about Hub internals                            │
└────────────────────┬────────────────────────────────────────────┘
                     │ chan TodoEvent
┌────────────────────▼────────────────────────────────────────────┐
│  hub.go (Hub)                                                   │
│  • pure fan-out: Subscribe / Unsubscribe / Broadcast            │
│  • knows nothing about HTTP, DB, or domain types                │
└────────────────────┬────────────────────────────────────────────┘
                     │ Broadcast(TodoEvent) called after DB write
┌────────────────────▼────────────────────────────────────────────┐
│  plugin.go mutation handlers (handleTodo, handlePatch, etc.)    │
│  • calls p.hub.Broadcast() with the DB-returned value           │
│  • knows nothing about who is listening                         │
└─────────────────────────────────────────────────────────────────┘
```

Each layer depends only on the one below it. The Hub has no knowledge of HTTP or
types; the SSE handler has no knowledge of state management; the React hook has
no knowledge of component structure.

## Edge Cases

| Scenario                       | Behaviour                                   |
| ------------------------------ | ------------------------------------------- |
| No clients connected           | `Broadcast` drops into the buffered         |
:                                : channel, is consumed by the next `Run()`    :
:                                : iteration, and finds no clients —           :
:                                : effectively a no-op.                        :
| Client too slow to consume     | Events are dropped for that client (channel |
:                                : `select` with `default`). The next          :
:                                : reconnect triggers an initial GET to        :
:                                : resync.                                     :
| Server restart                 | `EventSource` auto-reconnects. On           |
:                                : reconnect, the initial `fetchTodos`         :
:                                : `useEffect` (which runs on `ws` change)     :
:                                : will re-fire because the component          :
:                                : remounts. If it doesn't remount, the manual :
:                                : Refresh button covers this case.            :
| Multi-tab editing              | All tabs share the same SSE stream          |
:                                : (separate connections). Tab A's mutation    :
:                                : broadcasts to Tab B's connection — both     :
:                                : update simultaneously.                      :
| Event emitted before SSE write | The mutation's `p.hub.Broadcast` is called  |
:                                : only after a successful DB write, so the    :
:                                : state is always consistent with what the DB :
:                                : contains.                                   :
